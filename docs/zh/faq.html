常見問題

關於 Chromium 系瀏覽器在 138 版後強制停用及移除「終結內容農場」？

Chromium 規劃將套件系統升級為 Manifest V3 (MV3)，並逐步淘汰 Manifest V2 (MV2) 套件。（官方公告）

「終結內容農場」作為 MV2 套件，也會被強制停用及移除。解決方案如下：

法一：調整瀏覽器設定以允許繼續安裝和使用 MV2 套件

  1. 在 Chromium 系瀏覽器網址列輸入及造訪 chrome://flags。
  2. 尋找 #temporary-unexpire-flags-m137，設定為 Enabled。
  3. 重新啟動瀏覽器。
  4. 尋找 #extension-manifest-v2-deprecation-unsupported，設定為 Disabled。
  5. 尋找 #extension-manifest-v2-deprecation-disabled，設定為 Disabled。
  6. 重新啟動瀏覽器。

完成以上設定後，即可繼續安裝和使用「終結內容農場」。

注意：此為 Chromium 提供的過渡用設定，未來可能無法使用。

法二：安裝「終結內容農場」MV3 測試版

注意：MV3 需要做系統層級的設定才能發揮較完整的網路請求攔截功能（安裝時會跳出相關提示），功能性不如 MV2。為避免影響目前使用者，預計在 MV2 版本完全無法使用時才會更新為 MV3 正式版。

法三：改用非 Chromium 核心瀏覽器

這是最治本的方法。事實上，由於核心 API 實做差異，Firefox（或其他 Gecko 核心）瀏覽器上的「終結內容農場」才能發揮完整功能；Chromium 上的「終結內容農場」即使是 MV2 也存在一些功能缺陷（例如無法完全攔截啟動瀏覽器時載入的網頁）。

可否取消網站解鎖的等待時間和驗證碼？

「終結內容農場」目前沒有移除此功能的規劃，因為「終結內容農場」本質上就是藉由製造些許不便幫助使用者自律的工具。

抵制內容農場本來就不是輕鬆愉快的事——看到聳動的標題、摘要、貼圖時，你必須克制一窺究竟的衝動，花點時間讓頭腦冷靜，轉而尋找更合適的資料。如果沒有打算為抵制泛濫成災的內容農場付出一點心力及忍受些許不便，一開始就不需要安裝。

「終結內容農場」的設計理念是，當一個網站被加入黑名單，就是使用者想抵制的對象，因此「終結內容農場」會竭盡所能標示、提醒、避免讓使用者接觸；如有調查內容的需要，通常也可以用特製的安全 [檢視] 功能解決。[解鎖] 理當只須用於少數 [檢視] 無法正常運作的網頁，不會是常態。

如有頻繁解鎖的需求，請重新省思：是否根本不認為該網站是內容農場，或者出於其他理由不打算抵制該網站？若是如此，將該網站從黑名單移除或加入灰名單或白名單即可，這步驟並不需要花太多時間。

可否把列入黑名單的網站連結隱藏或更改樣式？

「終結內容農場」沒有這方面的計畫，主要包括倫理上及技術上的考量。

倫理上來說，目前並無絕對的標準可以定義一個網站是或不是內容農場——雖然我們為提供的名單設立了一套標準，但僅僅是我們基於現實考量選擇的一些我們認為對世人最有幫助的切入點而已，並非真理——也就無法保證黑名單絕對符合使用者的需求，而可能發生誤判，誤判的網站連結一但被隱藏，就可能導致有用的資訊被錯過。相較之下，目前的連結標示已有很好的提醒效果，且使用者可以自主判斷標示是否正確，我們認為這是比較理想的方式。

技術上來說，單純隱藏超連結會導致排版上少一塊東西，效果並不理想（可試用實做了此功能的 Ban Bad Websites 體驗看看）。一般所謂的隱藏超連結其實是連相關前後文一併隱藏，而每個網站架構不同，沒有通用演算法能可靠地計算出相關前後文範圍，又天下網站何其多，不可能一一客製。相較之下，「終結內容農場」目前的通用標示方法可適用幾乎所有網站。

如果你非常確定某些網站毫無價值，想眼不見為淨，也不會因此覺得可惜，還是可以搭配使用這類工具。這方面的工具我們推薦 uBlacklist，你可以個別加入確定要隱藏的網站，或直接訂閱我們提供的 uBlacklist 規格黑名單。
